Wellpad Survey/Follow-up Platformclose

This iPad-centric solution was commissioned in order to ameliorate an older system used for managing a health clinic's patient and staff data. It provides an intuitive survey platform, as well as an administrative interface for viewing current information and trends in both new and returning patients. My primary role throughout the duration of this project was to implement front-end views and components for the patient-intake and follow-up survey slides. Graphic design mock-ups of every view and component were issued out by team lead Lee Saynor, and I was to try and bring them to life and make them usable the best of my ability. Lead Developer Daniel Tabak maintained a unique and frenetic pace throughout every milestone.

The tech stack was and largely still is on the bleeding edge of web development today - ES6 React paired with Redux and styled with the Materialize CSS framework alongside Swiper providing cohesion and kinetics for the whole show (the latter 2 coincidentally being used on this very website). On the back-end, everything sat on top Node.js with data models being managed by MongoDB and Mongoose.js.

OpenDojo CMSclose

OpenDojo is a now open-sourced codebase that was implemented to create a content management system for a local karate club. Its features include a real-time reminder of the upcoming classes, a searchable/filterable student list with detailed individual profile views, belt rank and class creation views, a club-wide e-mail/announcement module, and a tablet-friendly interface for two-step logging-in for students attending classes.

Taking on multiple roles in this project, my contributions involved initial server and server routes setup with Node and Express.js, implementing a rudimentary front-to-back authentication module using JSON Web Tokens, maintaining thorough API and data flow documentation for the repository, as well as being largely responsible for the style guide and appearance of the front-end view elements.
